DOM Intro Última Actualización 15/12/2022

HTML DOM (Document Object Model) hace referencia al modelo de objetos del documento en el momento en que se carga una página web. Este modelo es construido a partir de un árbol de objetos, y cada objeto permite modelar tanto la ventana propia del navegador, la página web, el historial de navegación, así como cualquier elemento HTML. El DOM permite ser accedido a través de JavaScript, para poder crear, modificar, o eliminar elementos utilizando código de programación.


DOM o Document Object Model es un conjunto de utilidades específicamente diseñadas para manipular documentos XML. Por extensión, DOM también se puede utilizar para manipular documentos XHTML y HTML. Técnicamente, DOM es una API de funciones que se pueden utilizar para manipular las páginas XHTML de forma rápida y eficiente.


Antes de poder utilizar sus funciones, DOM transforma internamente el archivo XML original en una estructura más fácil de manejar formada por una jerarquía de nodos. De esta forma, DOM transforma el código XML en una serie de nodos interconectados en forma de árbol. El árbol generado no sólo representa los contenidos del archivo original (mediante los nodos del árbol) sino que también representa sus relaciones (mediante las ramas del árbol que conectan los nodos).


Aunque en ocasiones DOM se asocia con la programación web y con JavaScript, la API de DOM es independiente de cualquier lenguaje de programación. De hecho, DOM está disponible en la mayoría de lenguajes de programación comúnmente empleados.

Al estar "amparado" por un lenguaje de programación, todas estas tareas se pueden automatizar, incluso indicando que se realicen cuando el usuario haga acciones determinadas, como por ejemplo: pulsar un botón, mover el ratón, hacer click en una parte del documento, escribir un texto, etc...

Pero, ¿quién define la forma en la que DOM debe funcionar en los navegadores y motores web? Se trata del W3C o World Wide Web Consortium, que además maneja otros sistemas. El W3C es una entidad internacional, creada en 1994 por Tim Berners-Lee, el padre de la Internet; que desarrolla padrones abiertos, asegurando el crecimiento de la web. Dentro de la entidad existe un grupo llamado Web Platform Working Group, responsable por DOM. La W3C es responsable por diversos otros padrones, protocolos y lenguajes, como: CSS; HTML; XML; SVJ.

DOM Animaciones

JavaScript permite modificar y agregar animaciones de CSS a través de código que permite agregar dinamismo a un objeto HTML que una vez que ha sido renderizado al visualizar el sitio web se mantiene estático. Es posible volver a dibujar nuevamente el elemento utilizando JavaScript, sin embargo, se debe tener en cuenta que consume más recursos volver a dibujar el elemento en cada frame, y algunas animaciones más complejas pueden requerir una mayor capacidad de procesamiento del computador en el que se visualiza el elemento.

DOM Eventos

La interacción de JavaScript con HTML se maneja a través de eventos que ocurren cuando el usuario o el navegador manipula una página. Cuando se carga la página, se llama un evento. Cuando el usuario hace clic en un botón, ese clic también es un evento. Otros ejemplos incluyen eventos como presionar cualquier tecla, cerrar una ventana, cambiar el tamaño de una ventana, etc. Los desarrolladores pueden usar estos eventos para ejecutar respuestas codificadas en JavaScript, lo que hace que los botones cierren ventanas, se muestren mensajes a los usuarios, se validen datos y prácticamente cualquier otro tipo de respuesta imaginable.

Ejemplo

JavaScript

function saludo1() {
    alert("Grupo 03")
}

HTML

<form>
    <input type = "button" onclick = "saludo1()" value = "Dar click aquí" />
</form>

DOM Event Listener

Con el método .addEventListener() permite añadir una escucha del evento indicado (primer parámetro), y en el caso de que ocurra, se ejecutará la función asociada indicada (segundo parámetro). De forma opcional, se le puede pasar un tercer parámetro con ciertas opciones, que veremos más adelante:

Con .addEventListener() se pueden añadir fácilmente varias funcionalidades.

Con .removeEventListener() se puede eliminar una funcionalidad previamente añadida.

Con .addEventListener() se pueden indicar ciertos comportamientos especiales.

Ejemplo

JavaScript

document.getElementById("botonlistener").addEventListener("click", function(){
    document.getElementById("texto1").innerText = "El texto aparece con EventListener";
});

HTML

<button id="botonlistener">Click here</button>
<h1 id="texto1"></h1>

Navegación DOM

Los nodos en el árbol DOM siguen la estructura jerárquica entre sí.

El nodo que está en la parte superior del árbol DOM se conoce como el nodo raíz.

Cada nodo tiene un nodo principal excepto el nodo raíz

Un padre puede tener cualquier número de hijos.

Los nodos cuyos padres son iguales se llaman vecinos o hermanos.

Existen tres formas posibles de moverse a través de un árbol:

Hacia abajo en el árbol

Hacia arriba en el árbol

Hacia un lado en el árbol

Sintaxis: Al usar la propiedad del nodo firstChild en un nodo, obtendrá el primer nodo del elemento.
const primero = element.firstChild;

firstElementChild devuelve el primer hijo del padre que es un elemento.
var primerelemento = element.firstElementChild.nodeName;

lastElementChild devuelve el último hijo del padre que es un elemento.
var lastElement = element.lastElementChild.nodeName;

Utilice la propiedad lastChild para seleccionar el último elemento secundario del elemento principal.
var ultimo = element.lastChild.nodeName;

La propiedad child del elemento devuelve HTMLCollection que contiene todos los elementos secundarios del elemento especificado.
for (let i = 0; i < element.childNodes.length; i++) { console.log(element.childNodes[i]); }

Nodos DOM

Los nodos DOM proporcionan varias propiedades y métodos que permiten navegar o atravesar la estructura de árbol del DOM y realizar cambios muy fácilmente.

Puede usar las propiedades firstChild y lastChild del nodo DOM para acceder al primer y último nodo secundario directo de un nodo, respectivamente. Si el nodo no tiene ningún elemento secundario, devuelve nulo.

El DOM nos permite hacer cualquier cosa con sus elementos y contenidos, pero lo primero que tenemos que hacer es llegar al objeto correspondiente del DOM.

Todas las operaciones en el DOM comienzan con el objeto document. Este es el principal “punto de entrada” al DOM. Desde ahí podremos acceder a cualquier nodo.

Los nodos DOM están anidados unos dentro de otros, un elemento puede contener muchos otros elementos dentro de él, como se muestra en el ejemplo a continuación.

En este ejemplo se podrá visualizar la estructura de árbol que presenta DOM. Encontramos un nodo padre de tipo nav, con id navbar. Dentro del nodo padre tenemos un nodo hijo que es un ul para contener un elemento hijo, y finalmente, dentro del nodo ul podemos encontrar un nodo hijo de tipo li para un elemento de lista.

Colecciones DOM

Un objeto de colección HTML es una colección de elementos HTML de forma similar a un arreglo y representa a los elementos de un documento. 

    El índice comienza desde cero (0). Los elementos de la colección se ordenan en el orden en que aparecen en el código fuente.
    Contiene un número que representa el índice del elemento que el usuario quiere devolver. El índice comienza desde 0.
    Sintaxis: HTMLCollection.item(index)

El método getElementsByTagName() acepta un nombre de etiqueta y devuelve una colección HTML en vivo de elementos con el nombre de etiqueta coincidente en el orden en que aparecen en el documento.

                            Sintaxis: let elementos = document.getElementsByTagName(nombreEtiqueta);

Lista de Nodos DOM

Un objeto NodeList es esencialmente una colección de nodos (elementos DOM) tomados del documento HTML. Es muy parecido a una matriz, pero no admite métodos de matriz, ya que son dos cosas completamente diferentes.

    El método querySelectorAll() devuelve una lista de nodos estática, mientras que la propiedad Node.childNodes devuelve una lista de nodos activa.

                            
Cambiar una única propiedad CSS

El método NodeList values() devuelve un iterador que le permite recorrer todos los valores contenidos en el objeto NodeList. Los valores son objetos de nodo.

    Sintaxis:
    
    ListaNodos.valores();
    Parámetros: Este método no toma ningún parámetro.
    
    Valor devuelto: este método devuelve un iterador.
